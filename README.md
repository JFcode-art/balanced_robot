# STM32F407 平衡机器人

一个基于 STM32F407 + FreeRTOS 的两轮自平衡小车，带蓝牙遥控和 QT 上位机。

## 能做什么

- **自己站稳** — MPU6050 测姿态，互补滤波算角度，PID 控电机，倒了能自己起来
- **定速巡航** — 编码器测速，速度闭环，跑起来速度稳得住
- **定点停车** — 位置闭环，让它去哪就去哪，误差不大
- **蓝牙遥控** — 手机 or 电脑发指令，调速度、调角度、设位置
- **QT 上位机** — 串口连上就能看实时曲线、调参数

### 接线表

| STM32 引脚 | 接什么 | 干啥用 |
|------------|--------|--------|
| PA9, PA10 | USB 转 TTL | 调试口 |
| PA2, PA3 | 蓝牙模块 | 遥控通信 |

| PA0, PA1 | 左电机编码器 | TIM2 测速 |
| PB6, PB7 | 右电机编码器 | TIM4 测速 |
| PA6, PA7 | 左电机 PWM | TIM3 正反转 |
| PB0, PB1 | 右电机 PWM | TIM3 正反转 |

### PWM 和编码器不打架吗？

原始设计踩过这个坑 — TIM2 和 TIM4 又想当编码器又想输出 PWM，同一组定时器干不了两件事。现在的方案是：

- **TIM2、TIM4** 专职读编码器
- **TIM3** 专门输出 PWM（4 个通道，左右电机各占俩）

I2C 也挪到了 I2C2，因为 PB6/PB7 被 TIM4 编码器占用了，别硬挤。

## FreeRTOS 怎么跑的

开了 6 个任务，按优先级排：

| 任务 | 优先级 | 周期 | 干什么 |
|------|--------|------|--------|
| Sensor | 最高 | 1ms | 读 MPU6050 |
| Attitude | 次高 | 1ms | 互补滤波算角度 |
| Control | 高 | 10ms | 三环 PID 算 PWM |
| MotorStatus | 中高 | 20ms | 监测电机状态 |
| Bluetooth | 中 | 10ms | 收蓝牙指令、回传状态 |
| Comm | 最低 | 5ms | USB 串口收发 |

任务之间靠队列传数据，外加任务通知做唤醒信号，不搞全局变量乱飞。

### 通知链路

| 谁发通知 | 通知谁 | 啥时候发 | 收到通知干啥 |
|----------|--------|----------|-------------|
| Task_Control | Task_Comm | 每次算完新控制量 | 从队列取数据，发串口上报 |
| Task_MotorStatus | Task_Bluetooth | 每次更新完电机状态 | 从队列取状态，发蓝牙上报 |

Task_Comm 不再死等队列（portMAX_DELAY），而是等通知 + 5ms 超时，没通知时就轮询一下串口有没有收到上位机指令。Task_Bluetooth 同理，不用每次循环都去读一遍队列。

## PID 怎么调

三环从内往外调，顺序别搞反：

1. **角度环（最内层）** — 先调 Kp，从 10 开始往上加，加到车身能立住但有轻微抖动；再加 Kd 压住抖动。Kp 大概 15-25，Kd 大概 1-3。
2. **速度环（中间层）** — Kp 控制速度响应，Ki 消静差。Kp 大概 2-5，Ki 大概 0.1-0.5。
3. **位置环（最外层）** — Kp 控制位置精度，Ki 不要太大。Kp 大概 0.5-2，Ki 大概 0.01-0.1。

代码里的初始值：

```c
PID_Init(&pid_angle,    20.0f, 0.0f, 2.0f, 100.0f, -100.0f);
PID_Init(&pid_speed,     3.0f, 0.2f, 0.0f,  50.0f,  -50.0f);
PID_Init(&pid_position,  1.0f, 0.05f, 0.0f, 10.0f, -10.0f);
```

实际值看你车身的重量和重心高度，没有万能参数，得试。

## 通信协议

### USB 串口（115200）

上位机发命令给下位机：

```
[0xAA][指令码][参数高8位][参数低8位][0x55]
指令码：0x01=设速度，0x02=设角度，0x04=停止
```

下位机上报数据：

```
[0xAA][0x01][角度][速度][位置高][位置低][PWM高][PWM低][0x55]
```

### 蓝牙（9600）

帧格式统一：

```
[0xBB][数据长度][命令码][数据...][0xCC]
```

支持的命令：
- 0x01 设速度（2 字节，int16/100）
- 0x02 设角度（2 字节，int16/100）
- 0x03 设 PID（6 字节，Kp/Ki/Kd）
- 0x04 紧急停止
- 0x05 请求状态
- 0x06 设位置（2 字节，int16/10）

## 调参注意事项

- 角度环是基础，角度环没调好千万别动速度环和位置环
- 调角度环时把车拿手上悬空，感受电机响应，别一上来就放地上
- Ki 给大了容易积分饱和，车会突然抽风
- 第一次通电建议把 PWM 限幅设小一点（比如 ±500），别上来就满油门
- 电池电压掉到 10V 以下电机扭矩不够，车会站不稳

## 安全问题

- 电机功率不小，手指别靠近齿轮和车轮
- 测试时周围别有易碎品
- 建议装个物理急停开关，代码里的急停是软件层面的，万一死机就废了
- 上电前检查一下接线，正负极别接反
